# Product Review

代码审查记录目录。每次审查产出一份按"阶段 + 日期"命名的报告。

## 命名规范

```
{YYYY-MM-DD}-{phase}-review.md
```

例如：
- `2026-06-24-phase0-1-review.md` — Phase 0+1 联合审查
- `2026-07-XX-phase2-review.md` — Phase 2 Cambium Engine 审查

## 文档结构约定

每份审查报告应包含：

1. **审查范围** — 涵盖哪些模块 / 文件
2. **测试状态** — 测试通过情况 + 覆盖度
3. **整体评价** — spec 还原度、设计正确性、不变量保护
4. **Bug 列表** — 按 P0/P1/P2 分级
5. **改进建议** — 不阻塞但值得清理的项
6. **修复优先级建议** — 哪些必须先修、哪些可延后
7. **待确认的设计问题** — 不是 bug，需要决策
